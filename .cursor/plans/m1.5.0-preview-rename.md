# Feature Plan Lite: M1.5.0 — Rename plan workflow to preview

## Goal

Align the user-facing read-only workflow with `CONTEXT.md`: rename `plan` to `preview` across CLI, core workflow API, tests, and docs. Preserve the strict read-only invariant (preview never writes config, lockfiles, manifests, managed state, source locations, or Client Discovery Locations).

## Out of scope

- M1.5.1 (`sync` → `apply`) and later M1.5 tasks
- Renaming internal **planner** / structured **plan entry** terminology used for future M5 planning engine (distinct from the Preview workflow)
- `--upgrade` → `--refresh-sources` (separate glossary item)

## Decision: compatibility alias

**Remove `plan` CLI subcommand** — no temporary alias. Rationale: pre-V1 codebase with workflow stubs only; glossary explicitly avoids `plan` for the read-only workflow; alias would slow ubiquitous-language alignment.

## Implementation steps

### 1. Core workflow API (`crates/agentcfg-core/src/workflow.rs`)

- Rename `PlanRequest` → `PreviewRequest`, `PlanResult` → `PreviewResult`
- Rename `pub fn plan` → `pub fn preview`
- Update module doc comment: replace “planning” workflow wording with “preview” where it refers to the read-only workflow (keep “planning” only if referring to future planner subsystem)

### 2. CLI args (`crates/agentcfg-cli/src/args.rs`)

- Rename `CliCommand::Plan(PlanArgs)` → `CliCommand::Preview(PreviewArgs)`
- Rename struct `PlanArgs` → `PreviewArgs`
- Add clap subcommand name: `#[command(name = "preview")]` on the Preview variant (or equivalent so CLI is `agentcfg preview`)
- Keep `#[command(about = "Preview configured changes without writing them")]`
- Update tests: `agentcfg plan` → `agentcfg preview` in parse tests; reject `agentcfg plan` if no alias

### 3. CLI commands (`crates/agentcfg-cli/src/commands.rs`)

- Import and use `PreviewRequest`, `workflow::preview`
- Rename `WorkflowInvocation::Plan` → `Preview`
- Rename test `maps_plan_forms_to_workflow_request` → `maps_preview_forms_to_workflow_request`
- Update CLI argv in tests from `plan` to `preview`

### 4. Documentation (user-facing workflow only)

Update command examples and workflow names where they mean the read-only Preview workflow:

| File | Change |
|------|--------|
| `README.md` | `plan` command → `preview`; prose “plan is read-only” → “preview is read-only” |
| `docs/prd.md` | `agentcfg plan` examples and `plan` workflow bullets → `preview` |
| `docs/design-v1.md` | `agentcfg plan` table rows and examples → `preview`; section “Plan, Sync, and Prune” → “Preview, Sync, and Prune” where about workflows |
| `docs/implementation-plan-v1.md` | Mark M1.5.0 checkboxes done; note `plan` command renamed in M1 list where it lists CLI commands |

**Do not** mass-rename `planner`, `plan records`, `plan --upgrade` test names in *future* milestone tasks unless they are clearly the Preview CLI command in completed/current sections.

### 5. Validation

```sh
cargo test --workspace preview
cargo test --workspace
cargo run -p agentcfg-cli -- preview --help
```

Confirm `agentcfg plan` fails to parse (no alias).

## Acceptance criteria

- [ ] CLI subcommand is `preview` with same flags as former `plan` (`--user`, `--upgrade`)
- [ ] Core exports `PreviewRequest`, `PreviewResult`, `preview()`
- [ ] All tests pass; workspace tests filtered by `preview` match renamed tests
- [ ] Docs describe `preview` as the read-only workflow
- [ ] No `plan` subcommand or `PlanRequest` / `workflow::plan` remains in Rust code
- [ ] M1.5.0 items in `docs/implementation-plan-v1.md` checked off

## Risks

- Low: workflow is a stub; rename is mechanical
- Doc churn: avoid changing M5+ “planner” technical terms in this task
