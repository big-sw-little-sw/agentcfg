# Feature Plan Lite: M1.5.1 — Rename sync workflow to apply

## Goal

Align the user-facing write workflow with `CONTEXT.md`: rename `sync` to `apply` across CLI, core workflow API, tests, and docs. Preserve the one-way invariant: apply writes managed state and Client Discovery Locations, never source locations.

## Out of scope

- M1.5.2+ (Install Level, config layer wording, `--refresh-sources`, etc.)
- Renaming future milestone **locked_path_sync**, **sync_apply**, or other internal/sync-as-verb implementation task names in not-yet-implemented sections unless they refer to the **CLI** `sync` command in current user-facing docs
- `--upgrade` → `--refresh-sources` (M1.5.4)
- Mass-renaming "sync" in future M4/M6/M8 implementation task descriptions that mean synchronization behavior, not the Apply workflow name

## Decision: compatibility alias

**Remove `sync` CLI subcommand** — no temporary alias. Rationale: same as M1.5.0 (`plan` → `preview`); pre-V1 stubs; glossary explicitly avoids `sync` for the apply workflow.

## Implementation steps

### 1. Core workflow API (`crates/agentcfg-core/src/workflow.rs`)

- Rename `SyncRequest` → `ApplyRequest`, `SyncResult` → `ApplyResult`
- Rename `pub fn sync` → `pub fn apply`
- Update module doc comment if it says "sync" for the apply workflow (line ~4 mentions "apply" already in planning context — ensure workflow naming is consistent)

### 2. CLI args (`crates/agentcfg-cli/src/args.rs`)

- Rename `CliCommand::Sync(SyncArgs)` → `CliCommand::Apply(ApplyArgs)`
- Rename struct `SyncArgs` → `ApplyArgs`
- Add clap subcommand name: `#[command(name = "apply")]` on the Apply variant
- Keep `#[command(about = "Apply configured changes")]` (already present on Sync variant)
- Update parse tests: `agentcfg sync` → `agentcfg apply`
- Add rejection test: `agentcfg sync` should fail to parse (no alias), mirroring `agentcfg plan` rejection for preview

### 3. CLI commands (`crates/agentcfg-cli/src/commands.rs`)

- Import and use `ApplyRequest`, `workflow::apply`
- Rename `WorkflowInvocation::Sync` → `Apply`
- Rename test `maps_sync_forms_to_workflow_request` → `maps_apply_forms_to_workflow_request`
- Update CLI argv in tests from `sync` to `apply`

### 4. Documentation (user-facing Apply workflow)

Update command examples and workflow names where they mean the one-way Apply workflow:

| File | Change |
|------|--------|
| `README.md` | `sync` command → `apply`; prose about sync workflow → apply |
| `docs/prd.md` | `agentcfg sync` examples and `sync` workflow bullets → `apply` |
| `docs/design-v1.md` | `agentcfg sync` table rows and examples → `apply`; section "Preview, Sync, and Prune" → "Preview, Apply, and Prune" where about workflows; sync-as-workflow prose → apply |
| `docs/implementation-plan-v1.md` | Mark M1.5.1 checkboxes done; update M1.1 bullet to say `apply` (formerly `sync`) |

**Do not** rename future milestone internal test names like `locked_path_sync` or `sync_apply` in sections that describe not-yet-built behavior, unless the text clearly refers to the user-facing CLI command in completed/current milestones.

### 5. Validation

```sh
cargo test --workspace apply
cargo test --workspace
cargo run -p agentcfg-cli -- apply --help
```

Confirm `agentcfg sync` fails to parse (no alias).
