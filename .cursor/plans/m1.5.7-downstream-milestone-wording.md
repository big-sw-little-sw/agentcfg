# Feature Plan Lite: M1.5.7 — Update downstream milestone wording

## Goal

Update M2 and later milestones in `docs/implementation-plan-v1.md` to use terms from `UBIQUITOUS-LANGUAGE.md` before implementation agents start M2 work. Replace pre-glossary vocabulary in downstream task titles, checklists, validation commands, and end-to-end acceptance notes.

## Out of scope

- Rust/CLI code changes (no behavior change)
- Renaming completed M1.5 historical task descriptions that document prior renames
- Renaming internal core module names such as `planner` where design-v1 intentionally keeps a lower-level API boundary
- Updating `docs/prd.md`, `docs/design-v1.md`, or `README.md` beyond what downstream plan references require

## Term mapping (M2+)

| Avoid | Use |
|-------|-----|
| sync | apply |
| plan (workflow) | preview |
| upgrade | Source Refresh (`--refresh-sources`) |
| InstallScope / scope (install context) | Install Level / Config Layer |
| Consumer / consumers (domain) | Discovery Requirement / Discovery Requirements |
| installed name | Discovery Name |
| original name | Source Skill Name |
| target registry / client target | Client Discovery Registry / Client Discovery Location |
| Managed Source Tree / managed source | Managed Skill Content |
| target artifact | Installed Artifact |
| desired target state/artifacts | Desired State (structured entries for Installed Artifacts) |
| stale consumer / stale artifact | Stale Discovery Requirement / Stale Installed Artifact |
| unexpected target / broken target | Unexpected Symlink Target / Broken Symlink |
| alias (config feature) | Skill Alias |
| cache leftovers | unused Managed Skill Content / Managed State leftovers |

Keep serialized schema names (`resource_kind`, `target_path`, `consumers`, etc.) only where tasks explicitly note intentional implementation-only fields until schema migration.
