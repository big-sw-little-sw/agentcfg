# Feature Plan Lite: M1.5.6 — Align status, prune, and safety terminology

## Goal

Align ubiquitous language for **Unmanaged Artifact**, **Stale Discovery Requirement**, **Unsatisfied Discovery Requirement**, **Stale Installed Artifact**, **Unexpected Symlink Target**, **Broken Symlink**, **Status**, and **Doctor** across Rust APIs, CLI help, tests, and domain docs per `CONTEXT.md`. Remove user-facing "unmanaged installed artifact", "stale consumer", "broken target", and conflated diagnose/inspect wording where glossary terms apply.

## Out of scope

- M1.5.7 downstream milestone mass-rename
- Implementing status/prune/stale-state engines (M6/M7) — vocabulary and glossary anchors only
- Renaming `CliError::Unexpected` or other non-domain "unexpected" identifiers
- Renaming internal `DiscoveryLocationInspection` (filesystem scan, not Status workflow)
- Changing `doctor` rejecting `--user` behavior (already correct)

## Decision: Rust renames

**Rename `UnmanagedInstalledArtifact` → `UnmanagedArtifact`** (glossary: **Unmanaged Artifact**, not "unmanaged installed artifact").

| Old | New | Notes |
|-----|-----|-------|
| `UnmanagedInstalledArtifact` | `UnmanagedArtifact` | Init finds Manifest-unrecorded entries at Client Discovery Locations |
| `InitWarning::UnmanagedInstalledArtifact` | `InitWarning::UnmanagedArtifact` | |
| `unmanaged_installed_artifacts` test helper | `unmanaged_artifacts` | |
| `unmanaged_project_installed_artifacts_*` test | `unmanaged_project_artifacts_*` or include `unmanaged_artifact` | |

**Glossary anchor module** — add `crates/agentcfg-core/src/install_health.rs`:

| Anchor | Purpose |
|--------|---------|
| `status_glossary_*` test | **Status** reports managed install-state consistency |
| `prune_glossary_*` test | **Prune** removes **Stale Discovery Requirements** and **Stale Installed Artifacts** safely |
| `doctor_glossary_*` test | **Doctor** reports environment/configuration readiness (not install consistency) |
| `stale_discovery_requirement_glossary_*` | Manifest requirement no longer in Desired State |
| `unsatisfied_discovery_requirement_glossary_*` | Desired State requirement without valid Installed Artifact |
| `stale_installed_artifact_glossary_*` | Manifest artifact with no remaining Discovery Requirements |
| `symlink_diagnostic_glossary_*` | **Unexpected Symlink Target** / **Broken Symlink** scoped to symlink destinations only |

Register `mod install_health;` in `lib.rs`. No behavior change.

## Implementation steps

### 1. Core (`crates/agentcfg-core`)

**`workflow.rs`**

- Apply `UnmanagedArtifact` renames throughout init scan path
- Expand module doc: **Status** vs **Doctor**; **Prune** removes stale managed state using glossary stale terms
- Rename tests/helpers per table; ensure names match `cargo test status prune doctor` filters

**`install_health.rs`** (new)

- Module doc listing install-health glossary terms and Status/Doctor split
- `#[cfg(test)]` glossary constant tests as in `desired_state.rs`
- Test names must include substrings: `status`, `prune`, `doctor`, and at least one each for stale/unsatisfied discovery requirement, stale installed artifact, symlink diagnostics

### 2. CLI (`crates/agentcfg-cli`)

**`args.rs`**

- `prune` about: mention **Stale Installed Artifacts** and **Stale Discovery Requirements** (concise)
- `status` about: managed install-state consistency per **Status** glossary (avoid "inspect" as verb for status)
- `doctor` about: environment and configuration readiness (avoid implying install-state inspection)

**`commands.rs`**

- User-facing init warning: "unmanaged artifact" not "unmanaged installed artifact"

**`tests/init.rs`**

- Update stderr assertion for new warning text

### 3. Documentation

| File | Changes |
|------|---------|
| `docs/prd.md` | Add Terms: **Unmanaged Artifact**, **Stale Discovery Requirement**, **Unsatisfied Discovery Requirement**, **Stale Installed Artifact**, **Unexpected Symlink Target**, **Broken Symlink**; distinguish **Status** (managed install consistency) vs **Doctor** (environment readiness); replace "unmanaged installed artifact" / generic "stale managed artifacts" with glossary terms where describing prune/status behavior; keep symlink terms for symlink diagnostics only |
| `docs/design-v1.md` | "broken targets" → **Broken Symlink** / broken symlinks; "unexpected symlink targets" capitalization; stale consumer references in future-facing prose → **Stale Discovery Requirement** / **Stale Installed Artifact** where user-facing; Status vs Doctor sections aligned |
| `docs/implementation-plan-v1.md` | Check off M1.5.6 items |
| `README.md` | Prune/status/doctor bullets use glossary terms; symlink safety uses **Unexpected Symlink Target** |

Do not rename `target_path` in manifest examples reserved for symlink internals.

### 4. Validation


Confirm **Status** docs do not use "diagnose" for install-state reporting; **Doctor** docs do not claim to be **Status**.
