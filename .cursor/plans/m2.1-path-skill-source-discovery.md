## Goal

Implement bounded filesystem discovery for path **Skill Sources** under `skill_source/`, with per-source `discovery_depth`, nested-skill exclusion, and duplicate **Source Skill Name** detection. Resolve configured paths relative to the config file. Do not grow `workflow::init` with resolution logic.

## Decisions (from grill)

| Topic | Choice |
| --- | --- |
| Layout | Walk tree from resolved Skill Source directory up to `discovery_depth` path segments |
| Nested skills | Directory with `SKILL.md` is a **Skill**; do not scan its children (silent skip) |
| Depth config | `[[skill_sources]].discovery_depth`, default **4**, max **8**, per source only |
| Source Skill Name | Leaf directory name (organizational subdirs do not qualify the name) |
| Duplicate leaf names | Error listing all conflicting paths |
| Path in config | Relative (to config file parent) or absolute; resolve at discovery time |
| Hidden dirs | Skip directory entries whose name starts with `.` |
| `skills.toml` | Skill Source root only â€” **out of M2.1** (M2.3) |
| Empty Skill Source | `Ok([])` from discovery; CLI **warning** in M2.2+ workflow, not in core API |
| Glossary | No `CONTEXT.md` changes |

Layout open question: answered by bounded `discovery_depth` + nested-skill exclusion (see `design-v1.md` Â§ Skill Source Discovery).

## Discovery algorithm

Inputs: `config_file: &Path`, `skill_source_id`, configured `path`, `discovery_depth`.

1. `resolved_root = resolve_skill_source_root(config_file, path)`  
   - Absolute `path` â†’ use as-is  
   - Relative â†’ `config_file.parent().join(path)` (normalize with `Path` only; no `canonicalize` in M2.1)
2. If `resolved_root` missing, not a directory, or is a file â†’ `SkillSourceError::NotFound { skill_source_id, configured_path, resolved_path }`
3. Walk from `resolved_root`, carrying `depth` = number of path segments below `resolved_root` to the current directory
4. At each directory (skip if name starts with `.`):
   - If `SKILL.md` exists as a file in this directory â†’ record skill (including `depth == 0` at `resolved_root`); **do not** descend
   - Else if `depth < discovery_depth` â†’ descend into child directories only
   - Else â†’ stop descending this branch
5. Files and non-directory entries â†’ ignore
6. On `read_dir` / metadata errors â†’ return `Error::Io { path, source }` (no silent skip)
7. After scan: duplicate `source_skill_name` â†’ `SkillSourceError::DuplicateSourceSkillName { skill_source_id, source_skill_name, skill_dirs }` with sorted paths
8. Sort discovered skills by `source_skill_name` for deterministic output

**Depth example** (`discovery_depth = 4`): `a/b/c/foo/SKILL.md` has 4 segments â†’ OK; five segments â†’ not discovered.

**Symlinks:** No cycle detection in M2.1; cyclic directory graphs may recurse until stack overflow; revisit with visited set if needed.
